# APIの作成・管理

microCMSでは、コンテンツを管理するためのAPIを作成・管理することができます。APIは「リスト形式」と「オブジェクト形式」の2つの形式から選択できます。

## APIの作成

### 1. API作成画面に移動

管理画面で「APIを作成」をクリックします。

### 2. 基本設定

- **API名**: APIの名前を入力（例: blog, news, products）
- **API ID**: URLで使用される識別子（自動生成される）
- **APIの形式**: リスト形式またはオブジェクト形式を選択

### 3. フィールドの設定

APIに含めるフィールドを設定します：

- **テキストフィールド**: タイトル、名前など
- **テキストエリア**: 本文、説明など
- **リッチエディタ**: HTML形式のコンテンツ
- **画像フィールド**: 画像のアップロード
- **日時フィールド**: 公開日時など
- **セレクトフィールド**: カテゴリーなど
- **コンテンツ参照**: 他のAPIとの関連付け

## APIの形式

### リスト形式

複数のコンテンツを管理する場合に使用します。

**特徴:**
- 複数のコンテンツを作成可能
- 一覧表示が可能
- ページネーション対応

**使用例:**
- ブログ記事
- ニュース
- 商品一覧
- お客様の声

### オブジェクト形式

単一のコンテンツを管理する場合に使用します。

**特徴:**
- 1つのコンテンツのみ管理
- 設定ページや固定ページに適している

**使用例:**
- サイト設定
- 会社情報
- お問い合わせフォーム設定

## API設定

### 権限設定

APIキーごとに以下の権限を設定できます：

- **GET**: コンテンツの取得
- **POST**: コンテンツの作成
- **PATCH**: コンテンツの更新
- **DELETE**: コンテンツの削除
- **下書き全取得**: 下書きコンテンツの取得

### フィルター設定

APIで取得できるコンテンツを制限できます：

- **公開済みのみ**: 公開されたコンテンツのみ取得可能
- **下書きも含む**: 下書きコンテンツも取得可能

## APIスキーマの管理

### エクスポート

APIスキーマをJSONファイルとしてエクスポートできます。

### インポート

既存のAPIスキーマをインポートして、同じ構造のAPIを複製できます。
